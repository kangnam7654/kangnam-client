Data Flow: CLI ↔ GUI

사용자 입력부터 CLI 응답 렌더링까지의 흐름

메시지 흐름 (1회 대화 턴)
1
User → React Chat UI에 메시지 입력
2
Tauri IPC → invoke("cli_send_message", { provider, message, conversationId })
3
CLI Manager → stdin에 메시지 write (이미 실행 중인 subprocess에)
4
CLI Process → stdout로 JSON 스트림 출력 (줄 단위)
5
JSON Parser → CLI별 어댑터가 JSON을 UnifiedMessage로 변환
6
Tauri Event → emit("cli-stream", UnifiedMessage) → React에서 실시간 렌더링

UnifiedMessage 스키마

CLI별 JSON을 이 공통 포맷으로 변환

UnifiedMessage (Rust enum)
enum UnifiedMessage {
  // 에이전트가 텍스트를 출력 중
  TextDelta { text: String },

  // 도구 호출 시작 (파일 읽기, 편집, 검색 등)
  ToolUseStart { id: String, name: String, input: Value },

  // 도구 실행 결과
  ToolResult { id: String, output: String, is_error: bool },

  // 사용자 확인 요청 (파일 수정, 명령 실행 등)
  PermissionRequest { id: String, tool: String, description: String },

  // 턴 완료
  TurnEnd { usage: Option<TokenUsage> },

  // 에러
  Error { message: String },
}

CLI별 어댑터 패턴

Claude Code 어댑터
// Claude Code JSON 출력 예시
{"type":"assistant","message":
  {"content":[
    {"type":"text","text":"파일을 읽겠습니다"},
    {"type":"tool_use","id":"x","name":"Read",
     "input":{"file_path":"/src/main.rs"}}
  ]}}

       ↓ 변환

UnifiedMessage::TextDelta { "파일을 읽겠습니다" }
UnifiedMessage::ToolUseStart { "Read", ... }
Codex CLI 어댑터
// Codex CLI JSON 출력 예시
{"type":"message",
 "content":"코드를 수정하겠습니다"}
{"type":"function_call",
 "name":"edit_file",
 "arguments":{...}}

       ↓ 변환

UnifiedMessage::TextDelta { "코드를 수정..." }
UnifiedMessage::ToolUseStart { "edit_file", ... }

핵심 설계 결정

Long-running subprocess — 대화마다 새 프로세스가 아니라, 세션 동안 하나의 CLI 프로세스를 유지. stdin/stdout으로 대화 지속.
Permission 가로채기 — CLI가 사용자 확인을 요청하면(파일 수정 등), GUI에서 시각적 다이얼로그로 보여주고 stdin으로 응답 전달.
어댑터 trait — 새 CLI 추가 시 CliAdapter trait만 구현하면 됨. 파싱 로직만 다르고 나머지는 공유.
